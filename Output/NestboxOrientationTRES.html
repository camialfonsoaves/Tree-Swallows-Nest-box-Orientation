Nest orientation analysis in Tree Swallows

2026-07-15

Overview

This project investigates the relationship between nest box entrance orientaion and nest-box occupancy, clutch size, and hatching success in the Tree swallows. All analyses were conducted in R Core Team 2024 and the code included data preparation, model fitting, diagnostic test, and figure graphing.

ANALYSIS 1: Seasonal occupancy patterns

Objective

This analysis tests whether nest-box occupancy changes over the breeding season and whether this temporal pattern differs among nest entrance orientations.

Because preliminary linear analyses did not detect an effect of day of the year on nest-box occupancy, generalized additive models (GAMs) were used to evaluate potential non-linear relationships between occupancy and time.

The following steps prepare the dataset for analysis: 1. Convert variables to factors. 2.Generate date and day-of-year variables using lubridate. 3.Restrict the dataset to the main breeding period (15 April–1 June 2023). 4.Convert nest-box status into a binary occupancy variable.

tres <-read.csv("../Data/tres.csv")

tres$orientation <- as.factor(tres$orientation)
tres$nest.status <- as.factor(tres$nest.status)
tres$locality <- as.factor(tres$locality)
tres$box.num <- as.factor(tres$box.num)
tres$species <- as.factor(tres$species)

tres$date2 <- dmy(paste(tres$day,tres$month,tres$year))
tres$date2 <- as.Date(as.character(tres$date2), format = "%Y-%m-%d")
tres$doy <- yday(tres$date2) ## day of the year
tres$woy <- week(tres$date2)

tres2 <- tres[tres$date2 >= "2023-04-15" & tres$date2 <= "2023-06-01", ]

Nest-box status was converted into a binary occupancy variable (1 = active nest with eggs, chicks, or fledglings; 0 = inactive nest).

tres2$status2 = if_else(tres2$nest.status == "Available", 0,
                        if_else(tres2$nest.status == "Start", 0,
                        if_else(tres2$nest.status == "Medium", 0,
                        if_else(tres2$nest.status == "Complete", 0,
                        if_else(tres2$nest.status == "Eggs", 1,
                        if_else(tres2$nest.status == "Chicks", 1,
                        if_else(tres2$nest.status == "Fledged", 1,
                        if_else(tres2$nest.status == "Failed", 0, false=NA))))))))
tres2$status2 <- replace(tres2$status2,is.na(tres2$status2),0)

Model setup

A simple GAM describing seasonal occupancy was compared with a more complex model including nest orientation, locality, and random effects for nest box.

GAM_TRES <- gam(status2 ~ s(doy, k=-1),
                family = "binomial", 
                data=tres2, 
                method = "REML") # method: REML Restricted Maximum Likelihood

GAM_TRES2 <- gam(status2 ~ s(doy, by=orientation, k=-1)
                 + orientation # interaction time by orientation
                 + locality # fixed effect of locality
                 + s(box.num, bs="re"), # control random effect of boxes
                 family = "binomial", 
                 data=tres2, 
                 method = "REML")

Model summaries and interpretation

Average nest occupancy did not differ among entrance orientations (parametric effects), but occupancy changed significantly through the breeding season in a non-linear manner within each orientation (smooth terms).

ANALYSIS 2: Clutch size and orientation

Objective

This model tests whether the number of eggs laid per nest box is associated with nest-box orientation while controlling for laying date (Julian day) and accounting for differences among the three study locations This process contains: 1. We Import data and prepare variables for the analysis. 2.Standardize Julian day to improve model fitting and interpretation. 3. Set the order of orientation levels and use West as the reference category

dataeggs <- read.csv("../Data/dataeggs.csv")
hist(dataeggs$Number.eggs.laid)

dataeggs$Percentage.eggs.hatched <- as.numeric(dataeggs$Percentage.eggs.hatched)
dataeggs$First.egg <- mdy(dataeggs$First.egg)
dataeggs$julianfegg <- yday(dataeggs$First.egg)
dataeggs$Location <- as.factor(dataeggs$Location)
dataeggs$Box.number <- as.factor(dataeggs$Box.number)
dataeggs$Orientation <- as.factor(dataeggs$Orientation)
dataeggs$julian_z <- scale(dataeggs$julianfegg)
dataeggs$Orientation <- factor(dataeggs$Orientation, levels = c("North", "East", "South", "West"))

Linear models and possible GLMMS

This is a linear Model comparing the number of eggs laid within the four orientations. We also ran a Generalized Linear Mixed Model including fixed effects as Julian date and random effects as location. We set first “Orientation” as unordered categorical variable since compass directions are categories not rankings. North facing boxes are fixed as the reference because our hypothesis focus on south and east boxes.

dataeggs$Orientation <- factor(dataeggs$Orientation, ordered = FALSE) 
dataeggs$Orientation <- relevel(dataeggs$Orientation, ref = "North")
numbeggs <- lm(Number.eggs.laid~Orientation, data = dataeggs)
numbeggs2 <- lm(Number.eggs.laid~Orientation+julian_z, data = dataeggs)
numbeggs3 <- lm(Number.eggs.laid~Orientation+julian_z+Location, data = dataeggs)
glmmnumbegg <- lmer(Number.eggs.laid~Orientation+julian_z+(1|Location), data = dataeggs)

Evaluating model’s AIC and ANOVAs

We evaluate if models that included Julian date of first egg laid and location performed better than just clutch size and orientation. We also considered location as a random effect with a GLMM. The model with smaller AIC only included the relationship between clutch size and orientation controlled by Julian day of the first egg laid.

ANALYSIS 3: Hatching success and orientation

Objective

This model tests whether hatching success is associated with nest-box orientation. Hatching success is treated as a binary response, where a nest box is considered successful if at least one egg hatched and the chick was alive during nest checks. Nest boxes that were predated during the egg stage were excluded from this analysis.

Filtering database

Filtering the database to obtain hatching success.

dataeggs %>%
  filter(Predated_as_eggs=="0") -> hat.succ

Model set ups for hatching success

A binomial liner model of hatching success and orientation. Hatching success is defined as a binomial variable. Comparative models also include Julian day of the first egg laid and Location. Location is also considered as a random effect in a GLMMM

hatch1 <- glm(Hatched ~ Orientation, family = binomial, data = hat.succ)
hatch2 <- glm(Hatched ~ Orientation + julian_z, family = binomial, data=hat.succ)
hatch3 <- glm(Hatched ~ Orientation + julian_z + Location, family = binomial, data = hat.succ)
hatchmodel_glmm <- glmer(Hatched ~ Orientation + julian_z + (1 | Location),family = binomial, data = hat.succ)

Model comparison with ANOVAS and AIC values

Candidate models are compared using Akaike’s Information Criterion (AIC) and likelihood ratio tests (ANOVAs) to identify the most parsimonious model. The model including nest-box orientation and Julian date has the lowest AIC and is selected as the final model. Adding Location did not improve model fit neither is consider a significant random effect.
